C++ memory management

Que 1 - What is fragmentation in Heap?
Ans - As the data gets deleted from the heap it creates a little segments of freed up memory and other memory locations are still occupied. This is called fragmentation. 
If a new memory of the size that is empty on heap is required then that gets allocated. Actually the heap keeps it empty for that case only.
If a larger chunk of memory is required on heap that is not avaialble on heap the heap mananger then manages the memory and reallocates the fragraments and the already occupied memory. This can take large time.

Que 4 - How does the layout of the memory allocation looks like?
Ans - [allocator metadata/control block][usable memory][guard/check bytes/sentinel block]

Que 5 - What is the function of allocator metadata?
Ans - The allocator metadata tracks the memory size, if the previous block is free or occupied(previous block state). The allocator metadata is also necessary to free or delete the memory. 
Sometimes it also contains bits of alignmnet.
Following this header metadata is then usable memory where the data is stored.

Que 6 - What is there in the sentinel block?
Ans - The guard block is there for error detection. This block is there to detect overflow and memory corruption rather then memory leak.
This block tells us that someone wrote past the assigned memory.

Que 9 - Why maintaining a good virtual destrcutor chain matters?
Ans - Because if can leak memory. If a class is derived from any base class then the destructor should always have a virtual destructor in the front. 
If a virtual keyword is missing then the object created using that class will leak memory.
