Que 3 - How do you calculate the threadindex in a 1D system?
Ans - threadID = offset + threadId.x;
offset = blockDim.x * blockIdx.x;

Que 4 - How do you calculate the global threadid in a 2D system?
Ans - global_id = row_offset + block_offset + threadIdx.x;
row_offset = gridDim.x * blockDim.x * blockIdx.y;
block_offset = blockIdx.x * blockDim.x;

The variable gridDim is used for the size of the block and the blockDim is used for number of threads in a block. 

Que 6 - You have a 1D array of size N. The kernel is launched with a 1D grid and 1D blocks. Write the formula for the global thread index that each thread should use to access one element.
Ans - gid = blockIdx.x * blockDim.x + threadIdx.x;

Que 7 - You have a 1D array, but the kernel is launched with a 2D grid and 1D blocks. Each thread must still process exactly one element of the 1D array. How do you compute a single linear global index using blockIdx.x, blockIdx.y, and threadIdx.x?
Ans - gid = (blockIdx.y * gridDim.x + blockIdx.x)*blockDim.x + threadIdx.x;

Que 8 - You have a 2D matrix A of size M × N stored in row-major order. The kernel uses a 2D grid and 2D blocks. Each thread handles one element A[i][j]. Write expressions for i and j.
Ans - i = blockIdx.y * blockDim.y + threadIdx.y;
j = blockIdx.x * blockDim.x + threadIdx.x;

Que 9 - What would the grid and block dims look like in the main program?
Ans - threads=16, dim3 block(threads, threads), dim3 grid((N + threads - 1)/threads, (M + threads - 1)/threads).

Que 10 - Same 2D matrix M × N, but now the kernel is launched with a 1D grid and 1D blocks. Each thread still processes one matrix element. How do you compute i and j from a single global thread index?
Ans -  gid = blockIdx.x*blockDim.x + threadIdx.x;
i = gid/N
j = gid%N

Que 11 -  You have a 3D data volume of size Nx × Ny × Nz. The kernel uses a 3D grid and 3D blocks. Each thread processes one element. Write expressions for (i, j, k).
Ans - i = blockIdx.y * blockDim.y + threadIdx.y;
j = blockIdx.x * blockDim.x + threadIdx.x;
k = blockIdx.z * blockDim.z + threadIdx.z;

Que 16 - What is occupancy?
Ans - Occupancy = Active warps / Max allowed warps

Que 17 - What are the factors which decide the occupancy?
Ans - Registers used per thread. 
Shared memory of used per block. 
Granularity constraint.

Que 18 - Our kernel uses 48 registers per thread, 4096 bytes of SMem per block. Find the occupancy for GTX 1650. 
Ans - Lets say registers per SM = 65536, registers per warp = 32 * 48 = 1536, allowed warps = 65536/1536 = 42. But the Granularity is 4. Meaning 4 warps can be scheduled and no less. Hence 40 active warps. 
Shared memory per block = 98304 bytes. Active blocks = 98304/4096 = 24. Active warps = warps per sm * active blocks per SM = 96. So the max warps are limited by the registers used. 

max warps allowed = 64. Occupancy = 40/64 = 63%

Que 22 - How can theoretical occupancy decrease if the register pressure is more?
Ans - Suppose we have 65536 total registers per SM. Suppose we have 256 threads per block. Now lets assume each thread uses 32 registers. Then each block uses 32 * 256 = 8192 registers. 
Now each SM can have 65536/8192 = 8 number of blocks = 64 warps. 
Now let assume each thread uses 128 registers. Now each block will use 32768 registers. Which is half of the total registers on an SM. No of active blocks = 2. Active warps = 16. This is how the occupancy has reduced significantly. 

Que 23 - Lets do the same excercise with shared memory.
Ans - Suppose we have 48KB of shared memory. 256 threads in total. 6KB of memory is used per block. Then we can have active blocks = 8. Now if each block uses 24 KB of shared memory then we can have active blocks = 2.
